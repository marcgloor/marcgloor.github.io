This benchmark measures the raw computational throughput of a processor by executing a high-intensity floating-point loop across all available CPU cores. By utilizing Fused Multiply-Add (FMA) style operations, it estimates the theoretical raw Peak GFLOPS of the system.




TECHNICAL SPECIFICATIONS

* Parallelization: OpenMP (Automatic thread scaling)
* Operation Type: Single-precision Floating Point (float)
* Arithmetic: a = a * b + c (10 ops per iteration)
* Iterations: 1,000,000,000 (Defined as ITERATIONS)
* Cache Target: L1 Cache (Minimal memory footprint to avoid RAM bottlenecks)


COMPILATION

Use a compiler with OpenMP support (GCC/Clang recommended).

Command:
  gcc -O3 -fopenmp flops.c -o flops

*Note: -O3 is essential for the compiler to vectorize the arithmetic and pipelining correctly.*

Run the binary:  ./flops

The program will display:
- Detected CPU Core count
- Total execution time
- Calculated MFLOPS/GFLOPS


CAUTION

This test generates significant heat as it pushes all CPU cores to 100% load. Ensure your cooling system is operational!
